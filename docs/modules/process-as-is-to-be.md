## 1. AS-IS 프로세스 (v1 현재 상태)

### 1-1. 구현된 컴포넌트

| 컴포넌트 | 파일 | 역할 |
|---|---|---|
| Library | `src/components/Library/Library.tsx` | 분야별 고전 텍스트 목록, 난이도 표시, 텍스트 선택 |
| Practice — TextLayer | `src/components/Practice/PracticeTextLayer.tsx` | 원문 텍스트 표시·하이라이팅 |
| Practice — Canvas | `src/components/Practice/PracticeCanvas.tsx` | Cytoscape 기반 노드-엣지 그래프 캔버스 (단일 모드) |
| Practice — Toolbar | `src/components/Practice/PracticeToolbar.tsx` | 노드 타입 선택·엣지 생성 도구 |
| Practice — Evaluation | `src/components/Practice/EvaluationPanel.tsx` | 세션 자가 평가 입력 |
| Chat (AI 튜터) | `src/components/Chat/ChatPanel.tsx` | Claude/GPT/Gemini/Ollama/Kimi 멀티 프로바이더 채팅 |
| Feedback | `src/components/Feedback/FeedbackPanel.tsx` | 학생 피드백 제출 + 전체 피드백 열람 (`/api/feedback`) |
| TextLayer | `src/components/TextLayer/TextLayer.tsx` | (v1 독립 텍스트 레이어, Practice 외부 노출용) |
| VisualLayer | `src/components/VisualLayer/VisualLayer.tsx` | Cytoscape 시각화 레이어 독립 컴포넌트 |
| PatternPanel | `src/components/PatternPanel/PatternPanel.tsx` | 인지 패턴 카테고리 (인지/가치/시간) 선택 패널 |

### 1-2. v1 학습 사이클 (AS-IS)

```
[Library] ── 텍스트 선택 ──→ [Practice]
                                ├── [TextLayer] ← 원문 열람
                                ├── [Canvas]    ← 노드 그래프 작성 (단일 모드, 캔버스 저장 없음)
                                ├── [Toolbar]   ← 노드/엣지 조작
                                └── [PatternPanel] ← 인지/가치/시간 카테고리
                                        │
                                        ▼
                              [AI 튜터 Chat] ← 실시간 질문/피드백
                                        │
                                        ▼
                              [EvaluationPanel] ← 세션 종료 자가 평가
                                        │
                                        ▼
                              [FeedbackPanel] ← 제출 → /api/feedback
```

### 1-3. 미구현 영역 (v1의 공백)

| 영역 | 상태 | 근거 |
|---|---|---|
| 인증 (이메일 인증, 소셜 로그인) | ❌ 없음 | n300, n301 신규 |
| 온보딩 (튜토리얼, 3축 소개) | ❌ 없음 | n302, n303 신규 |
| 구독/결제 | ❌ 없음 | n320, n321 신규 |
| 캔버스 모드 선택 (자유형/제약형/단계형) | ⚠️ assumption | n262/n263/n274가 assumption 노드 → n304로 전환 |
| 캔버스 저장/갤러리/다운로드 | ❌ 없음 | n305, n306, n322 신규 |
| AI 튜터 프롬프트 관리 | ❌ 없음 | n307, n319 신규 |
| 응답 품질 평가 (별점) | ❌ 없음 | n308 신규 |
| 튜터 대화 히스토리 | ❌ 없음 | n309 신규 |
| 학습 완료 평가 → 성장 리포트 | ⚠️ 부분 | EvaluationPanel 있으나 집계·리포트 없음 |
| 3축 분석 리포트 | ❌ 없음 | n311, n312 신규 |
| 알림 센터 / 리마인더 | ❌ 없음 | n313, n314 신규 |
| 관리자 확장 (통계/그룹/콘텐츠/모듈) | ❌ 없음 | n315–n319, n323 신규 |
| DB 기반 사용자 계정 | ❌ 없음 | v1은 이름 입력 수준 (n182) |
| 학습 진도 추적 (대시보드) | ⚠️ 없음 | n205, n206은 스펙에만 존재 |

### 1-4. v1 데이터 흐름

- **콘텐츠**: 정적 JSON (`src/data/texts`) → Library → useStore
- **캔버스 상태**: `usePracticeStore` (인메모리, 새로고침 소실)
- **AI 채팅**: `/api/chat` (Express) → 멀티 프로바이더 SDK 직접 호출
- **피드백**: `/api/feedback` POST/GET (Express + 파일 또는 메모리)
- **시간 비용 큰 자리**: 캔버스 상태 비영속 (매 세션 재작업), 피드백 집계 수동

## 2. TO-BE 프로세스 (v2)

### 2-1. 9개 신규 카테고리별 사용자 여정

#### 카테고리 A. 인증 강화

**노드**: n300(이메일인증), n301(소셜로그인)

```
[회원가입 n193]
  ├── 이메일/비밀번호 가입 → [이메일 인증 n300] → 인증 완료 → [대시보드 n205]
  └── 소셜 로그인 [n301: Google/Kakao OAuth] → [대시보드 n205]

[비밀번호 찾기 n194] → 이메일 발송 → [재설정 n220]
```

**데이터**: EmailToken(purpose='verify'), OAuthAccount, Lucia v3 세션
**시간 비용**: 인증 메일 발송 ≈ 즉시, 토큰 만료 24h

---

#### 카테고리 B. 온보딩

**노드**: n302(튜토리얼), n303(방법론 소개)

```
[대시보드 n205] ─ 첫 로그인 감지 → [온보딩 튜토리얼 n302]
  └── [Brain180 방법론 소개 n303: 인지/가치/시간 3축 설명]
        └── 완료 → [대시보드 n205] (온보딩 완료 플래그 기록)
```

**데이터**: User.onboardedAt (nullable), 튜토리얼 완료 플래그
**반복 빈도**: 최초 1회

---

#### 카테고리 C. 구독/결제

**노드**: n320(플랜선택), n321(결제처리)

```
[회원가입 n193] ─ 요금제 선택 → [구독 플랜 n320: free/standard/premium]
  └── 유료 선택 → [결제 처리 n321: 카드/카카오페이 토스페이먼츠]
        └── 결제 완료 → [이메일 인증 n300] → [대시보드 n205]

무료 플랜 → [이메일 인증 n300] 직행
```

**데이터**: Plan, Subscription, Payment, TossBillingKey
**핸드오프**: 류한길[워크플로] — 결제 webhook 상태기계 설계 필요

---

#### 카테고리 D. 캔버스 모드 결정 (assumption → 기능)

**노드**: n304(모드선택: 자유형/제약형/단계형) — n262/n263/n274 해소

```
[연습 페이지 n225] → 캔버스 진입
  └── [캔버스 모드 선택 n304]
        ├── 자유형(free): 노드/엣지 제약 없음, Konva 자유 캔버스
        ├── 제약형(constrained): 허용 노드 타입·엣지 수 제한
        └── 단계형(guided): 순서 있는 빌드 가이드 + 힌트
              └── 작업 → [저장/갤러리 n305] ↗
```

**기술**: Konva.js + react-konva (v1 Cytoscape 대체 또는 병행)
**핸드오프**: 백그림[UI/UX] — 3모드 분기 UI 와이어 필요

---

#### 카테고리 E. 캔버스 저장/갤러리/다운로드

**노드**: n305(저장/불러오기), n306(갤러리), n322(다운로드)

```
[캔버스 작업] → [저장/불러오기 n305]
  ├── [갤러리 n306: 이전 회차 열람] → 불러오기 → 캔버스 복원
  └── [다운로드 n322: PDF/이미지 내보내기] → Cloudflare R2 업로드 → 서명 URL 반환
```

**데이터**: CanvasArtifact(payload JSON + thumbnailUrl), CanvasExport(format, url)
**시간 비용**: 저장 ≈ DB write + R2 업로드; 다운로드 ≈ R2 서명 URL 즉시

---

#### 카테고리 F. AI 튜터 관리

**노드**: n307(프롬프트관리), n308(품질평가), n309(히스토리), n319(품질관리)

```
[AI 튜터 채팅 n241] → 대화 세션
  ├── [응답 품질 평가 n308: 별점/코멘트] → [튜터 응답 저장 n256] → [AI 품질 관리 n319]
  └── [튜터 대화 히스토리 n309: 누적 대화 열람]

[관리자 대시보드 n243] → [AI 프롬프트 관리 n307: 시스템 프롬프트 버전 관리]
  └── [AI 품질 관리 n319: 응답 검토/프롬프트 개선]
```

**데이터**: TutorSystemPrompt(버전 관리), TutorMessage, TutorRating, APIUsageLog
**핸드오프**: 차곡담[데이터설계] — TutorSystemPrompt 버전 전환 스키마 설계

---

#### 카테고리 G. 평가/리포트

**노드**: n310(완료평가), n311(성장리포트), n312(3축분석)

```
[연습 페이지 n225] ─ 세션 종료 → [학습 완료 평가 n310: 자가 평가]
  └── [개인 성장 리포트 n311: 누적 학습 데이터 시각화]

```

**데이터**: SessionEvaluation, GrowthReport(axisCognitive/Value/Time Score), ExportCSV/PDF
**시간 비용**: 리포트 집계 = pg-boss 백그라운드 잡 (세션 종료 후 비동기)

#### 카테고리 H. 알림 시스템

[대시보드 n205] → [알림 센터 n313: 튜터 응답/진도 알림 열람]

## 3. 3개 액터 Swimlane

```
┌─────────────────────────────────────────────────────────────────────────────┐
│ 학생 (Student)                                                              │
│  가입/로그인 → 구독 → 이메일인증 → 온보딩 → 대시보드                        │
│  → 라이브러리 → 모듈선택 → 연습(텍스트+캔버스+도구)                         │
│  → AI튜터 채팅 → 튜터 품질평가 → 히스토리 열람                             │
│  → 세션완료평가 → 성장리포트 → 3축분석 → 통계내보내기(본인)                 │
│  → 알림센터 → 리마인더설정 → 캔버스 갤러리/다운로드                         │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │ API 호출 (인증 세션)
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ AI 튜터 (Anthropic Claude + OpenAI fallback)                               │
│  시스템 프롬프트 수신 → 학생 컨텍스트(현재 맵, 노드/엣지) 수신              │
│  → 응답 생성 → TutorMessage 저장 → APIUsageLog 기록                        │
│  → (학생) 품질평가 수신 → TutorRating 저장                                 │
│  → (관리자) 프롬프트 버전 업데이트 수신 → 다음 세션에 적용                  │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │ 관리 인터페이스
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 관리자 (Admin, role='admin')                                               │
│  관리자 로그인(동일 /login, role 분기) → 관리자 대시보드                    │
│  → 가입현황 조회 → 승인/보류 처리                                           │
│  → 학생 질문 관리 → 튜터 응답 검토                                         │
│  → AI 프롬프트 관리 → 품질 관리(개선 루프)                                 │
│  → 통계 대시보드 → 그룹/클래스 관리                                        │
│  → 콘텐츠 업로드 → 모듈 관리(순서/잠금)                                    │
│  → API 사용량 조회 → 학습 통계 내보내기(전체)                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 4. 핸드오프 포인트

### → 차곡담 [데이터설계]

| 자리 | 전달 사항 |
|---|---|
| TutorSystemPrompt | 버전 전환 트랜잭션: isActive 플래그 전환 시 진행 중 세션 처리 전략 필요 |
| CanvasArtifact | payload JSON 스키마 확정 필요 (Konva serialize 포맷) |
| GrowthReport | 3축 점수 집계 로직 — SessionEvaluation → axisCognitive/Value/TimeScore 매핑 |
| Subscription + Payment | 결제 webhook 수신 후 Subscription 상태 전환 (active/cancelled/expired) |
| ReminderRule → Notification | 잡 실행 결과를 Notification 테이블에 insert하는 스키마 연결 |

### → 류한길 [워크플로/상태기계]

| 자리 | 전달 사항 |
|---|---|
| 결제 상태기계 | pending → paid → active → expired / cancelled — Toss webhook 이벤트 매핑 |
| 캔버스 세션 상태기계 | 모드 선택 → 작업 중 → 저장/종료 → 평가 전환 흐름 |
| 리마인더 잡 | ReminderRule 기반 cron 스케줄 → push/email 발송 → Notification insert |
| 가입 승인 워크플로 | 가입 신청 → 관리자 검토 → 승인(이메일 발송) / 보류 — 상태 전환 명세 |

### → 백그림 [UI/UX]

| 자리 | 전달 사항 |
|---|---|
| 캔버스 3모드 UX | 자유형/제약형/단계형 진입 분기 화면 — 모드별 툴바·힌트 차이 |
| 온보딩 플로우 | n302(튜토리얼) → n303(3축 소개) 2단계 페이지 와이어 |
