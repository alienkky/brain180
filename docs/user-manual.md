# Brain180 학습 매뉴얼

> 천재의 글을 읽는 것이 아니라, 천재의 뇌로 보는 것.

---

## 프로그램 목표

Brain180은 고전 텍스트를 단순히 읽는 것이 아니라,  
저자가 **어떻게 생각했는지**(인지 구조)를 시각화하여  
깊은 독해 능력을 기르는 학습 프로그램입니다.

---

## 학습 흐름 요약

```
① 연습 모드: 텍스트를 읽고 직접 인지 구조 다이어그램을 만든다
        ↓
② 평가: 내가 만든 구조를 시스템이 채점하고 피드백을 준다
        ↓
③ 분석 모드 - 인지 구조: 시스템의 해석과 내 해석을 비교한다
        ↓
④ 분석 모드 - 가치 구조: 개념에 담긴 가치의 흐름을 확인한다
        ↓
⑤ 분석 모드 - 시간축: 사고의 시간적 전개 순서를 확인한다
```

## 화면 구성

### 상단 헤더

| 요소 | 설명 |
|---|---|
| **연습 버튼** | 연습 모드로 전환 (직접 다이어그램 만들기) |
| **분석 버튼** | 분석 모드로 전환 (시스템 해석 확인) |
| **인지 구조 / 가치 구조 / 시간축** | 분석 모드에서만 나타남. 세 가지 관점 전환 |
| **우측 텍스트** | 현재 학습 중인 텍스트의 저자와 제목 |

## 1단계: 연습 모드 — 인지 구조 만들기

연습 모드의 화면은 세 영역으로 나뉩니다:

| 영역 | 위치 | 역할 |
|---|---|---|
| 텍스트 패널 | 왼쪽 (32%) | 원문 텍스트에서 핵심 구절 선택 |
| 캔버스 | 중앙 | 노드 다이어그램 제작 공간 |
| 도구 패널 | 오른쪽 (22%) | 노드 역할, 연결어, 편집 도구 |

### 1-1. 텍스트에서 핵심 구절 선택하기

텍스트를 읽으면서 중요하다고 생각하는 단어/구절을 선택합니다.

#### 단어 하나 선택 (동그라미)
- 단어를 **탭(클릭)** 하면 빨간 동그라미가 생깁니다.
- 동그라미 친 단어를 다시 탭하면 해제됩니다.

#### 여러 단어 묶기 (구절)
- **방법 1 — Shift 클릭**: 첫 단어를 클릭한 뒤, 마지막 단어를 `Shift + 클릭`하면 그 범위가 하나의 구절로 묶입니다.
- **방법 2 — 꾹 누르기**: 첫 단어를 **길게 누르면**(0.5초) 묶기 모드에 진입합니다. 노란 안내가 나타나면 끝 단어를 탭합니다.
- **방법 3 — 묶기 버튼**: 하단의 `묶기` 버튼을 누른 뒤, 시작 단어와 끝 단어를 차례로 탭합니다.

#### 상태 표시
- 하단에 `구: N | 단어: N` 형태로 현재 선택한 구절 수와 단어 수가 표시됩니다.

### 1-2. 캔버스에 노드 추가하기

동그라미 친 구절을 캔버스의 노드로 보냅니다.

#### 추가 방법
- **↗ 버튼**: 동그라미 오른쪽에 나타나는 작은 `↗` 버튼을 클릭하면 즉시 캔버스에 추가됩니다.
- **드래그 앤 드롭**: 동그라미를 마우스로 잡아서 캔버스 영역에 놓습니다.

#### 노드 역할 설정
노드를 추가하기 전에, 도구 패널에서 **노드 역할**을 선택합니다:

| 역할 | 색상 | 의미 |
|---|---|---|
| **핵심** | 빨강 | 텍스트의 중심 사상. 전체 논지를 관통하는 핵심 개념 |
| **기둥** | 청록 | 핵심을 지탱하는 주요 개념. 핵심과 직접 연결되는 뼈대 |
| **다리** | 보라 | 개념과 개념을 논리적으로 이어주는 연결 고리 |
| **가지** | 파랑 | 핵심에서 파생되거나 부수적인 개념 |

> 팁: 먼저 텍스트에서 가장 중요한 개념(핵심)을 찾고,  
> 그것을 설명하는 개념(기둥)을 찾은 뒤,  
> 이들을 잇는 논리(다리)와 부수 개념(가지)을 배치하세요.

### 1-3. 노드 간 연결 만들기

개념들 사이의 관계를 선으로 표현합니다.

#### 연결 방법
1. 도구 패널에서 **잇기(↗)** 도구를 선택합니다.
2. **시작 노드**를 클릭합니다 (노란 점선 테두리가 나타남).
3. **도착 노드**를 클릭합니다.
4. 두 노드 사이에 화살표 연결이 생성됩니다.

#### 연결어 설정
연결(엣지)을 만들 때 연결어(라벨)를 붙일 수 있습니다:
- 도구 패널의 **연결어** 섹션에서 미리 선택하거나 직접 입력합니다.
- 텍스트에 등장하는 연결어 목록이 역할과 함께 표시됩니다.
- 연결어를 선택한 뒤 노드를 잇으면 해당 라벨이 자동으로 붙습니다.

#### 연결 편집
- 캔버스에서 연결선을 **클릭**하면 도구 패널에 편집기가 나타납니다.
- 연결어를 수정하거나 삭제할 수 있습니다.

### 1-4. 노드 편집 및 관리

#### 이미 만든 노드의 역할 변경
- 캔버스에서 노드를 클릭하면 도구 패널에 **선택된 노드 편집기**가 나타납니다.
- 핵심/기둥/다리/가지 중 역할을 변경할 수 있습니다.

#### 캔버스 도구

| 도구 | 아이콘 | 기능 |
|---|---|---|
| **선택** | ↖ | 노드를 클릭하여 정보 확인. 드래그로 위치 이동 |
| **잇기** | ↗ | 두 노드 사이에 연결선을 생성 |
| **삭제** | ✕ | 노드 또는 연결선을 클릭하여 삭제 |

#### 기타 기능
- **⊞ 맞춤 버튼**: 모든 노드가 화면 안에 보이도록 자동 조정합니다.
- **핀치/스크롤**: 캔버스를 확대/축소할 수 있습니다.
- **초기화**: 도구 패널 하단의 `초기화` 버튼으로 모든 노드와 연결을 삭제합니다.

## 2단계: 평가 받기

노드를 2개 이상 만들면 도구 패널의 **내 인지 구조 평가받기** 버튼이 활성화됩니다.

### 평가 항목

| 항목 | 비중 | 설명 |
|---|---|---|
| 개념 매칭 | 60% | 내가 선택한 개념이 시스템의 핵심 개념과 일치하는지 |
| 역할 매칭 | 40% | 노드에 부여한 역할(핵심/기둥/다리/가지)이 적절한지 |
| 연결 정확도 | 별도 | 내가 만든 연결이 시스템의 관계와 일치하는지 |

### 평가 결과 읽기

| 등급 | 점수 | 의미 |
|---|---|---|
| 뛰어남 | 80% 이상 | 저자의 인지 구조를 정확히 파악 |
| 양호 | 60~79% | 대부분의 핵심을 포착, 일부 보완 필요 |
| 발전 중 | 40~59% | 기본 구조 파악, 더 깊은 분석 필요 |
| 시작 단계 | 40% 미만 | 텍스트를 다시 읽고 핵심을 찾아보세요 |

### 피드백 활용
- **잘한 점**: 정확히 파악한 부분을 확인합니다.
- **조언**: 놓친 개념, 잘못된 역할, 부족한 연결에 대한 구체적 안내를 따릅니다.
- **개념 매칭 상세**: 각 노드별 점수를 확인하여 약한 부분을 파악합니다.

> 평가 후 안내: "분석 모드에서 가치 구조와 시간축 관점도 비교해 보세요."

## 3단계: 분석 모드 — 인지 구조 확인

상단의 **분석** 버튼을 누르고, **인지 구조** 관점을 선택합니다.

### 화면 구성

| 영역 | 위치 | 내용 |
|---|---|---|
| 텍스트 레이어 | 왼쪽 | 원문 텍스트에 구절별 색상 하이라이트 |
| 비주얼 레이어 | 중앙 | 시스템이 추출한 인지 구조 다이어그램 |
| 패턴 패널 | 오른쪽 | 사고 패턴, 인지 레이어 |

### 텍스트-다이어그램 연동
- 텍스트의 구절을 **클릭**하면 다이어그램에서 해당 노드가 **하이라이트**됩니다.
- 다이어그램의 노드를 **클릭**하면 관련 없는 노드가 흐려집니다.
- 구절 위에 **마우스를 올리면** 해당 개념의 설명이 나타납니다.

### 인지 레이어
오른쪽 패널 하단에 인지 레이어가 표시됩니다:

| 레이어 | 의미 |
|---|---|
| L1 표층 | 텍스트 표면에 직접 드러나는 개념들 |
| L2 중층 | 표층 개념들을 변환시키는 촉매적 개념 |
| L3 심층 | 변환 후 도달하는 깊은 의미의 개념들 |

### 사고 패턴
시스템이 발견한 저자의 반복적 사고 패턴이 카드로 표시됩니다.  
각 패턴 카드에는 관련 노드 태그가 있어 클릭하면 다이어그램에서 확인할 수 있습니다.

### 내 연습과 비교하기
- 내가 만든 다이어그램과 시스템의 다이어그램을 비교합니다.
- 놓친 개념, 다르게 분류한 역할, 빠뜨린 연결을 확인합니다.
- 왜 시스템이 그렇게 분류했는지 패턴 설명을 읽으며 이해합니다.

## 4단계: 분석 모드 — 가치 구조 확인

상단에서 **가치 구조** 관점으로 전환합니다.

### 가치 구조란?
같은 텍스트라도 **저자가 어떤 가치를 중시하는지** 관점에서 보면  
전혀 다른 구조가 드러납니다.

### 가치 유형

| 가치 | 색상 | 의미 |
|---|---|---|
| 진리 | 파랑 | 참과 거짓, 객관적 사실 |
| 아름다움 | 핑크 | 미적 가치, 조화 |
| 선 | 초록 | 도덕적 가치, 옳고 그름 |
| 자유 | 노랑 | 구속으로부터의 해방, 자율성 |
| 사랑 | 빨강 | 관계, 헌신, 정서적 유대 |
| 힘 | 회색 | 권력, 통제, 영향력 |
| 지혜 | 보라 | 통찰, 깊은 이해 |
| 유대 | 청록 | 연결, 소속, 공동체 |

### 확인하기
- 다이어그램의 노드 색상이 가치 유형에 따라 변합니다.
- 텍스트의 구절 하이라이트도 가치 색상으로 바뀝니다.
- 오른쪽 패널에 **가치 패턴**과 **가치 유형별 노드 목록**이 표시됩니다.
- 각 노드 위에 마우스를 올리면 **가치 설명**(왜 이 가치인지)이 나타납니다.

### 학습 포인트
- 인지 구조에서 "핵심"이었던 개념이 가치 관점에서는 어떤 가치를 담고 있는지 파악합니다.
- 저자가 어떤 가치를 대립시키고, 어떤 가치를 지향하는지 읽어냅니다.

---

## 5단계: 분석 모드 — 시간축 확인

상단에서 **시간축** 관점으로 전환합니다.

### 시간축이란?
저자의 사고가 **어떤 순서로 전개되는지** 보는 관점입니다.  
논리적 흐름, 인과관계, 변환의 단계를 시간 순서로 배열합니다.

### 시간 단계

| 단계 | 색상 | 의미 |
|---|---|---|
| 1단계 — 전(Before) | 회색 | 변환 이전의 초기 상태 |
| 2단계 — 촉매(Catalyst) | 노랑 | 변환을 일으키는 핵심 사건/행위 |
| 3단계 — 후(After) | 초록 | 변환 후 도달하는 새로운 상태 |

### 확인하기
- 다이어그램이 좌 → 우 방향으로 시간 순서대로 배치됩니다.
- 같은 단계의 노드는 같은 열에 정렬됩니다.
- 오른쪽 패널에 **시간 흐름 타임라인**이 표시됩니다.
- 각 단계 번호를 클릭하면 해당 단계의 노드가 강조됩니다.

### 학습 포인트
- "전 → 촉매 → 후" 패턴이 저자의 논리 전개에서 어떻게 작동하는지 이해합니다.
- 촉매(2단계)가 무엇인지 파악하는 것이 핵심입니다.

## 학습 사이클 반복

```
  ┌─────────────────────────────────────────────┐
  │                                             │
  │   ① 연습: 텍스트 → 내 인지 구조 만들기      │
  │                  ↓                          │
  │   ② 평가: 채점 + 피드백                     │
  │                  ↓                          │
  │   ③ 분석-인지: 시스템 해석과 비교            │
  │                  ↓                          │
  │   ④ 분석-가치: 가치 흐름 확인               │
  │                  ↓                          │
  │   ⑤ 분석-시간: 사고 전개 순서 확인          │
  │                  ↓                          │
  │   ⑥ 다시 연습: 이해를 바탕으로 재구성       │
  │                                             │
  └─────────────────────────────────────────────┘
```

한 텍스트에 대해 이 사이클을 2~3회 반복하면,  
저자의 사고 구조를 입체적으로 이해하게 됩니다.

## 노드 역할 가이드: 어떻게 분류할 것인가

### 핵심(root) 찾기
- 이 텍스트를 한 문장으로 요약한다면, 어떤 개념이 중심인가?
- 다른 모든 개념이 이 개념으로 수렴하는가?
- 이 개념을 빼면 텍스트의 의미가 무너지는가?

### 기둥(anchor) 찾기
- 핵심 개념을 설명하거나 지탱하는 주요 개념은?
- 핵심만큼은 아니지만, 없으면 구조가 약해지는 개념은?

### 다리(bridge) 찾기
- 두 개념 사이를 논리적으로 이어주는 것은?
- "A이므로 B이다"에서 "이므로"에 해당하는 개념은?

### 가지(branch) 찾기
- 핵심에서 파생되었지만 독립적으로도 성립하는 개념은?
- 예시, 보충 설명, 결론의 확장에 해당하는 개념은?

## 단축키 및 조작법

| 동작 | 방법 |
|---|---|
| 단어 선택/해제 | 단어 클릭 |
| 구절 묶기 | Shift+클릭 또는 길게 누르기 |
| 캔버스 확대/축소 | 마우스 스크롤 또는 핀치 |
| 캔버스 이동 | 빈 영역 드래그 |
